# 抖音UI定制器 - 文档版本注册表
本文档用于统一管理项目中所有文档的版本信息，确保文档版本的一致性和可追溯性。

## 文档版本规范

- 版本号遵循语义化版本规范 `MAJOR.MINOR`（如 1.0, 1.1 等）
- 主版本号 `MAJOR` 变更：不兼容的文档结构变更
- 次版本号 `MINOR` 变更：向下兼容的功能添加或修改
- 所有文档必须包含标准 YAML front matter 元数据
- 文档版本与项目版本保持独立管理

## 版本更新流程
1. 更新文档内容和版本号
2. 同步更新文档的 YAML front matter 元数据
3. 在本注册表中更新对应条目
4. 提交 Pull Request 进行审核
5. 合并后更新项目文档版本
6. 发布新的文档版本（如 1.1）

## 版本号更新规则
### 主版本号更新 (MAJOR)

- 文档结构发生重大变更
- 添加新的文档章节或删除现有章节
- 修复严重文档错误
- 项目背景或范围发生变化

### 次版本号更新 (MINOR)

- 更新文档内容或示例
- 添加新的功能说明或教程
- 修复小的文档错误
- 优化文档格式和可读性
