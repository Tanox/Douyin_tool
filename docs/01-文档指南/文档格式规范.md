# 抖音Web端界面UI定制工具 - 文档格式规范

## 1. 文档元信息

所有文档必须包含标准的YAML front matter，位于文档顶部：

```yaml
---
version: 1.0
last_updated: 2025-11-08
author: SutChan
changelog: "变更说明"
---
```

**字段说明：**
- `version`：文档版本号，遵循语义化版本规范（如1.0、1.1.2）
- `last_updated`：最后更新日期，格式为YYYY-MM-DD
- `author`：文档作者
- `changelog`：本次更新的主要变更内容

## 2. 标题层级

### 2.1 标题级别

- **一级标题（H1）**：`# 文档标题` - 每个文档只能有一个一级标题，作为文档主标题
- **二级标题（H2）**：`## 章节标题` - 用于文档的主要章节划分
- **三级标题（H3）**：`### 子章节标题` - 用于章节内的细分内容
- **四级标题（H4）**：`#### 详细内容标题` - 用于更具体的内容细分

### 2.2 标题命名规范

- 标题应简洁明了，准确反映内容
- 使用中文命名，避免中英文混杂
- 标题首字母大写
- 避免使用标点符号（除必要的顿号和引号外）

## 3. 文本格式

### 3.1 字体与字号

- 正文使用默认字体和字号
- 重要概念或术语可以使用**粗体**标注
- 强调内容可以使用*斜体*标注
- 代码或命令行使用反引号 `` ` `` 包裹

### 3.2 段落规范

- 每个段落之间空一行
- 段落首行不缩进
- 单行长度建议不超过80个字符，便于阅读
- 句子之间使用一个空格分隔

## 4. 列表格式

### 4.1 无序列表

```markdown
- 列表项1
- 列表项2
  - 子列表项
  - 子列表项
- 列表项3
```

### 4.2 有序列表

```markdown
1. 第一步
2. 第二步
   1. 子步骤1
   2. 子步骤2
3. 第三步
```

### 4.3 列表规范

- 列表项内容与标记之间空一个空格
- 列表项末尾不添加标点符号（除非内容是完整句子）
- 子列表使用两个空格缩进

## 5. 代码块格式

### 5.1 行内代码

使用反引号包裹：`function example()`

### 5.2 代码块

使用三个反引号并指定语言：

```javascript
function example() {
  console.log('Hello world');
}
```

### 5.3 代码规范

- 代码块必须指定正确的语言类型
- 代码应保持良好的缩进和格式
- 重要的代码块应有必要的注释说明

## 6. 表格格式

```markdown
| 列名1 | 列名2 | 列名3 |
|-------|-------|-------|
| 内容1 | 内容2 | 内容3 |
| 内容4 | 内容5 | 内容6 |
```

### 6.1 表格规范

- 表头行与内容行之间必须有分隔线
- 列与列之间使用管道符（|）分隔
- 对齐方式：默认为左对齐，可使用冒号指定对齐方式
  - 左对齐：`|:----|`
  - 居中对齐：`|:----:|`
  - 右对齐：`|----:|`

## 7. 链接与引用

### 7.1 内部链接

```markdown
[链接文本](文件路径)
```

**示例：**
```markdown
[用户指南](../06-开发指南/用户指南.md)
```

### 7.2 外部链接

```markdown
[链接文本](URL)
```

### 7.3 引用块

```markdown
> 引用内容
> 可以有多行
```

## 8. 图片格式

```markdown
![图片描述](图片路径)
```

**图片规范：**
- 所有图片应使用相对路径
- 图片应包含有意义的描述文本
- 图片大小应适当，避免过大
- 优先使用SVG格式图片

## 9. 文档结构

### 9.1 标准文档结构

1. YAML front matter（必要）
2. 文档标题（H1，必要）
3. 简介/概述（可选）
4. 目录（可选，适用于长文档）
5. 主体内容（使用H2-H4组织）
6. 参考资料（可选）
7. 附录（可选）

### 9.2 文档命名规范

- 使用中文命名文件
- 文件名应简洁明了，反映文档内容
- 文件名中可使用连字符（-）分隔多个词
- 避免使用特殊字符和空格

## 10. 特殊格式标记

### 10.1 注意事项

```markdown
> **注意：** 需要特别注意的内容
```

### 10.2 警告信息

```markdown
> **警告：** 可能导致严重后果的操作提示
```

### 10.3 提示信息

```markdown
> **提示：** 有用的提示和建议
```

## 11. 版本管理规范

- 文档更新后必须更新`last_updated`日期
- 内容有实质性变更时升级版本号
- 重大变更应在`changelog`中详细说明
- 文档版本应与代码版本保持同步

## 12. 审核与发布流程

1. 文档编写完成后进行自查
2. 提交给相关人员审核
3. 审核通过后更新版本信息
4. 发布到文档系统

## 13. 维护责任

- 文档所有者负责定期更新和维护文档
- 代码变更应同步更新相关文档
- 文档存在问题时应及时修复
