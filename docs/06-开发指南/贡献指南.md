## 开始贡献

### 报告问题

如果您发现问题或有功能建议，请先检查[GitHub Issues](https://github.com/SutChan/douyin_tool/issues)是否已有相关报告。如果没有，可以创建新的Issue描述您的问题或建议。

### 修复问题或添加功能

1. Fork本项目到您的GitHub账户
2. 克隆Fork后的仓库到本地
3. 创建新的分支进行开发
4. 提交您的修改
5. 创建Pull Request

### 开发分支规范

- `main`: 主分支，包含稳定版本代码
- `develop`: 开发分支，包含最新开发代码
- `feature/xxx`: 功能开发分支
- `bugfix/xxx`: Bug修复分支

## 开发环境搭建

### 必备环境

- Node.js 14.x 或更高版本
- npm 6.x 或更高版本
- Git

### 安装步骤

1. 克隆仓库
   ```bash
   git clone https://github.com/SutChan/douyin_tool.git
   cd douyin_tool
   ```

2. 安装依赖
   ```bash
   npm install
   ```

3. 构建项目
   ```bash
   npm run build
   ```

   构建完成后，您可以在`dist`目录中找到生成的用户脚本文件`douyin_ui_customizer.user.js`。

## 代码规范

### JavaScript 规范

- 使用ES6+语法
- 变量和函数使用小驼峰命名法 `camelCase`
- 常量使用全大写并下划线分隔 `UPPER_SNAKE_CASE`
- 类名使用大驼峰命名法 `PascalCase`
- 使用双引号包裹字符串
- 每行代码不超过100个字符
- 函数使用JSDoc注释

### CSS 规范

- 使用BEM命名规范（Block Element Modifier）
- 避免使用ID选择器
- 使用Flexbox或Grid布局
- 为关键样式添加注释

### 文档规范

- 使用Markdown格式
- 为所有公共API添加JSDoc注释
- 保持文档与代码同步

## 提交规范

### 提交信息格式

```
类型: 简短描述

详细描述（可选）
- 修改项1
- 修改项2

关联的Issue编号（如 Closes #123）
```

### 提交类型

- **feat**: 添加新功能
- **fix**: 修复bug
- **docs**: 更新文档
- **style**: 代码风格修改（不影响功能）
- **refactor**: 代码重构（不添加新功能或修复bug）
- **perf**: 性能优化
- **test**: 添加或修改测试
- **build**: 构建配置修改
- **ci**: CI配置修改

### 提交信息示例

```
feat: 添加暗黑主题支持

- 实现暗黑主题切换功能
- 更新主题配置管理
- 添加暗黑主题样式

```

## 问题反馈

当您遇到问题时，请提供以下信息：

- 浏览器类型和版本
- Tampermonkey/Greasemonkey版本
- 抖音UI定制器版本
- 问题的详细描述
- 复现步骤
- 相关截图（可选）

## Pull Request 流程

1. 确保您的分支基于最新的`develop`分支
2. 运行构建和测试确保代码可以正常工作
3. 创建Pull Request并填写详细信息
4. 等待代码审查
5. 根据审查意见进行修改
6. 合并到`develop`分支
