# 抖音Web端界面UI定制工具 - 详细开发指南

本文档为抖音Web端界面UI定制工具的详细开发指南，旨在帮助开发者理解项目结构、开发流程和最佳实践。

## 项目概述

抖音Web端界面UI定制工具是一个基于油猴脚本的工具，允许用户自定义抖音Web端的界面元素，包括短视频页面和直播间页面的各种UI组件。

## 开发环境准备

### 基本要求

- Node.js 12.x 或更高版本
- npm 6.x 或更高版本
- Git
- 支持油猴扩展的浏览器（Chrome/Edge/Firefox）

### 环境配置

1. 克隆项目仓库：
   ```bash
   git clone https://github.com/SutChan/douyin_tool.git
   ```

2. 进入项目目录：
   ```bash
   cd douyin_tool
   ```

3. 安装依赖：
   ```bash
   npm install
   ```

4. 构建项目：
   ```bash
   npm run build
   ```

## 项目结构详解

```
src/
  ├── main.js         # 主脚本入口，包含核心功能实现
  ├── config.js       # 配置管理模块，处理配置的保存和加载
  ├── ui_manager.js   # UI管理器模块，负责界面元素查找和定制
  ├── styles/
  │   ├── default.css # 默认主题样式
  │   └── dark.css    # 暗黑模式样式
  └── utils/
      ├── dom.js      # DOM操作工具函数
      └── storage.js  # 本地存储工具函数
dist/                 # 构建输出目录
docs/                 # 项目文档目录
build.js              # 构建脚本（支持自动版本号递增）
package.json          # 项目配置和依赖
install.html          # 一键安装页面
```

## 模块功能说明

### main.js

主脚本入口，负责：
- 初始化整个应用
- 注入样式
- 监听页面变化
- 创建浮动设置按钮
- 注册油猴菜单命令
- 处理版本更新检查

### config.js

配置管理模块，负责：
- 定义默认配置结构
- 处理配置的加载和保存
- 配置合并和验证
- 配置导入导出功能

### ui_manager.js

UI管理器模块，负责：
- 查找和操作页面元素
- 应用UI定制设置
- 创建设置面板
- 处理用户交互事件

### 样式文件

- `default.css`: 定义设置面板的默认样式
- `dark.css`: 定义暗黑模式下的设置面板样式

### 工具函数

- `dom.js`: 提供DOM操作的工具函数，如元素查找、样式修改等
- `storage.js`: 处理本地存储操作，封装localStorage的使用

## 开发流程

### 1. 代码修改

在修改代码前，建议先了解项目的基本结构和工作原理。修改时应遵循以下原则：

- 保持代码风格一致
- 添加必要的注释
- 确保向后兼容性
- 遵循模块化开发理念

### 2. 本地测试

修改完成后，可以通过以下步骤进行本地测试：

1. 运行构建命令生成最新的脚本：
   ```bash
   npm run build
   ```

2. 在Tampermonkey中安装生成的脚本文件
   - 打开Tampermonkey扩展
   - 点击"添加新脚本"
   - 复制`dist/douyin_ui_customizer.user.js`的内容
   - 粘贴并保存

3. 打开抖音网页版测试功能

### 3. 代码提交

提交代码时，请遵循以下规范：

- 使用有意义的提交信息
- 每个提交只包含一个功能或修复
- 提交前进行代码审查

### 4. 版本发布

当功能开发完成或bug修复后，可以发布新版本：

1. 更新`package.json`中的版本号
2. 运行构建命令：`npm run build`
3. 创建GitHub Release并上传构建产物

## 代码规范

### 命名约定

- 变量和函数名使用驼峰命名法（camelCase）
- 常量使用全大写加下划线（UPPER_SNAKE_CASE）
- 私有变量和函数使用下划线前缀

### 代码风格

- 缩进使用2个空格
- 行尾不添加分号（可选）
- 字符串使用单引号
- 花括号与代码在同一行
- 运算符两侧添加空格

## 最佳实践

### 性能优化

- 尽量减少DOM操作
- 使用事件委托处理大量元素事件
- 避免重复查询DOM元素
- 合理使用节流（throttle）和防抖（debounce）

### 兼容性考虑

- 考虑不同浏览器的兼容性
- 处理页面结构变化的情况
- 确保脚本在不同的抖音页面结构下都能正常工作

### 错误处理

- 添加适当的错误处理和日志记录
- 捕获可能的异常并提供友好的错误提示
- 确保脚本不会影响抖音页面的正常功能

## 调试技巧

- 使用浏览器的开发者工具查看控制台输出
- 使用`debugger`语句设置断点
- 在Tampermonkey中开启调试模式
- 使用油猴脚本的日志功能记录信息

## 常见问题排查

### 脚本不生效

- 检查Tampermonkey是否已启用
- 确认脚本的匹配模式是否正确
- 查看浏览器控制台是否有错误信息

### 页面元素找不到

- 抖音页面结构可能已更新，需要修改元素选择器
- 使用延时或轮询方式等待元素加载
- 考虑使用更通用的选择器策略

### 性能问题

- 检查是否有重复执行的代码
- 优化DOM操作，减少重排和重绘
- 检查是否有内存泄漏问题
