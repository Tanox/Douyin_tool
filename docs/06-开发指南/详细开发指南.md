# 抖音UI定制器 - 详细开发指南

本文档将详细介绍抖音UI定制器的开发流程、代码结构和最佳实践，帮助开发者快速上手并参与项目开发。

## 项目概述

抖音UI定制器是一个基于Web技术开发的浏览器用户脚本，用于自定义抖音网页版的界面外观和布局。项目采用现代化的JavaScript开发技术，支持主题切换、元素控制和自定义配置等功能。

## 开发环境要求

### 系统要求
- Node.js 14.x 或更高版本
- npm 6.x 或更高版本
- Git
- 现代浏览器（推荐Chrome/Edge/Firefox）

### 环境搭建步骤

1. 克隆项目仓库
   ```bash
   git clone https://github.com/SutChan/douyin_tool.git
   ```

2. 进入项目目录
   ```bash
   cd douyin_tool
   ```

3. 安装依赖
   ```bash
   npm install
   ```

4. 构建项目
   ```bash
   npm run build
   ```

## 项目目录结构

```
src/
├── main.js         # 入口文件，负责初始化和协调各模块
├── config.js       # 配置管理模块，处理默认配置和用户设置
├── ui_manager.js   # UI管理模块，负责创建和管理用户界面
├── styles/
│   ├── default.css # 默认主题样式
│   └── dark.css    # 暗黑主题样式
└── utils/
    ├── dom.js      # DOM操作工具函数
    └── storage.js  # 本地存储工具函数
dist/                 # 构建输出目录
docs/                 # 项目文档目录
build.js              # 构建脚本
package.json          # 项目依赖配置
install.html          # 安装说明页面
```

## 核心模块说明

### main.js

入口文件，负责：
- 初始化项目环境
- 加载配置信息
- 创建UI管理器
- 绑定事件监听器
- 处理页面加载完成事件
- 启动应用程序

### config.js

配置管理模块，负责：
- 定义默认配置项
- 处理用户配置的加载和保存
- 提供配置访问接口
- 验证配置的合法性

### ui_manager.js

UI管理模块，负责：
- 创建应用程序界面
- 管理主题切换
- 处理用户交互事件
- 更新页面元素状态

### 样式文件

- `default.css`: 定义默认主题的样式
- `dark.css`: 定义暗黑主题的样式，基于默认主题进行覆盖

### 工具函数

- `dom.js`: 提供常用的DOM操作工具函数，如元素选择、属性操作等
- `storage.js`: 封装本地存储操作，支持配置的持久化存储

## 开发流程

### 1. 代码编写

在开始编写代码前，请确保：
- 了解项目的目标结构和模块划分
- 熟悉JavaScript ES6+语法
- 遵循项目的编码规范
- 使用适当的注释说明代码功能

### 2. 构建与测试

开发完成后，执行以下步骤构建和测试：

1. 构建项目
   ```bash
   npm run build
   ```

2. 安装到浏览器
   - 打开Tampermonkey扩展
   - 点击"添加新脚本"
   - 导入`dist/douyin_ui_customizer.user.js`文件
   - 保存并启用脚本

3. 测试功能
   - 打开抖音网页版
   - 测试开发的功能是否正常工作

### 3. 代码提交

代码开发完成并测试通过后，执行以下步骤提交：
- 使用Git提交代码到本地仓库
- 编写详细的提交信息，说明修改内容
- 推送到远程仓库

### 4. 版本发布

修复bug或添加新功能后，执行以下步骤发布新版本：

1. 更新版本号
2. 更新CHANGELOG.md
3. 执行构建
4. 创建GitHub Release
5. 上传构建产物

## 编码规范

### JavaScript编码规范

- 遵循JavaScript标准风格，使用ES6+语法
- 使用2个空格进行缩进
- 语句结束必须使用分号
- 字符串使用双引号("")，除非嵌套需要
- 命名规范：
  - 变量/函数: 小驼峰命名法 `camelCase`
  - 类/构造函数: 大驼峰命名法 `PascalCase`
  - 常量: 全大写并使用下划线分隔 `UPPER_SNAKE_CASE`

### 注释规范

- 使用JSDoc格式注释所有函数
- 复杂逻辑添加行内注释
- 每个模块文件开头添加模块说明
- 使用中文进行注释

### 错误处理

- 关键操作使用try-catch进行错误处理
- 使用`logger.js`模块进行日志记录
- 提供友好的错误提示给用户

## 性能优化

- 使用防抖(debounce)和节流(throttle)优化事件处理
- 延迟加载非关键资源
- 最小化DOM操作频率
- 使用文档碎片优化批量DOM操作
- 避免不必要的重绘和回流

## 测试建议

- 测试功能的边界条件
- 测试性能和响应速度

## 常见问题

### 如何调试脚本？

- 在Tampermonkey中启用调试模式
- 使用浏览器的开发者工具设置断点
- 查看控制台输出的日志信息

### 如何添加新主题？

1. 在`src/styles/`目录下创建新的CSS文件
2. 在`config.js`中添加主题配置
3. 在`ui_manager.js`中添加主题切换逻辑
4. 构建并测试新主题

2. 在合适的模块中添加功能代码

5. 构建并测试新功能
