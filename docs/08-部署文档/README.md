# 部署文档

## 部署概述

Douyin Tool作为一个浏览器用户脚本，部署主要涉及构建和分发过程。本文档详细介绍如何构建和部署该工具。

## 构建过程

### 前置要求

- Node.js (v14+)
- npm或yarn

### 构建步骤

1. **安装依赖**

```bash
cd douyin_tool
npm install
```

2. **执行构建**

```bash
node build.js
```

3. **构建产物**

构建完成后，产物将生成在`dist`目录中，主要包括：

- `douyin_ui_customizer.user.js` - 主用户脚本文件

## 部署方式

### 1. 用户脚本管理器安装

用户可以通过以下步骤安装Douyin Tool：

#### 方法一：直接安装

1. 确保已安装用户脚本管理器（如Tampermonkey、Violentmonkey或Greasemonkey）
2. 访问脚本分发平台或项目GitHub页面
3. 点击"安装"按钮
4. 在用户脚本管理器中确认安装

#### 方法二：手动安装

1. 下载构建好的用户脚本文件
2. 打开用户脚本管理器的仪表板
3. 点击"添加新脚本"或"导入"
4. 粘贴脚本内容或选择下载的文件
5. 保存并启用脚本

### 2. 本地开发调试部署

对于开发者，可以采用以下方式进行本地调试：

1. 修改源代码
2. 运行构建脚本
3. 在浏览器中重新加载用户脚本
4. 访问抖音网页版测试修改

## 分发渠道

### 推荐的用户脚本分发平台

- **Greasy Fork** - 主流的用户脚本分享平台
- **OpenUserJS** - 开源用户脚本平台
- **GitHub Pages** - 通过项目页面提供直接下载

### 分发注意事项

- 确保脚本包含适当的元数据头信息
- 提供清晰的安装说明和使用指南
- 定期更新脚本版本
- 收集用户反馈并持续改进

## 版本发布流程

### 发布前准备

1. 更新`package.json`中的版本号
2. 更新`CHANGELOG.md`中的版本说明
3. 确保所有测试通过
4. 准备发布公告

### 发布步骤

1. 构建最新版本
2. 更新GitHub仓库代码
3. 创建GitHub Release
4. 更新分发平台上的脚本
5. 发布公告

## 回滚计划

如果发布的版本出现严重问题，应执行以下回滚步骤：

1. 在分发平台上恢复到上一个稳定版本
2. 发布回滚公告，说明问题和解决措施
3. 分析问题原因并修复
4. 重新构建并发布修复版本
