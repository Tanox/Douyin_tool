# 抖音Web页面UI定制工具 - 文档格式规范指南

## 1. 文档元数据规范
文档必须在开头包含YAML front matter格式的元数据，用于记录文档的基本信息。
```yaml
---
version: 1.0
last_updated: 2024-06-18
author: Sut
changelog: "更新文档内容，修复格式问题"
---
```

**元数据字段说明：**
- `version`: 文档版本号，采用语义化版本号格式 x.y.z，初始版本为1.0.0
- `last_updated`: 文档最后更新日期，格式为YYYY-MM-DD
- `author`: 文档作者名称
- `changelog`: 文档变更记录，简要说明本次更新的内容

## 2. 标题层级规范
### 2.1 标题层级结构

- **一级标题**（#）：仅用于文档的主标题
- **二级标题**（##）：用于主要章节的标题
- **三级标题**（###）：用于章节内的子部分标题
- **四级标题**（####）：用于更细粒度的内容分类

### 2.2 标题文本格式要求

- 标题文本前后不应有空格
- 避免标题文本的大小写规范（中文无大小写问题）
- 标题应简洁明了
- 避免在标题中使用特殊字符（如引号、括号等），除非必要

## 3. 文本格式规范

### 3.1 强调文本

- 使用星号包裹文本实现斜体强调：`*斜体文本*`
- 使用双星号包裹文本实现粗体强调：`**粗体文本**`
- 使用三星号包裹文本实现粗斜体：`***粗斜体文本***`
- 强调内容应与普通文本之间没有空格

### 3.2 引用格式
- 使用大于号标注引用内容
- 引用内容应缩进
- 多级引用可以使用多个大于号
- 引用内的文本可以使用其他Markdown语法

## 4. 列表格式规范

### 4.1 无序列表

```markdown
- 列表项1
- 列表项2
  - 子列表项1
  - 子列表项2
- 列表项3
```

### 4.2 有序列表

```markdown
1. 第一项
2. 第二项
   1. 子项1
   2. 子项2
3. 第三项
```

### 4.3 列表格式要求
- 列表项与内容之间必须有空格
- 列表项内容超过一行时，应保持缩进对齐
- 子列表应相对于父列表缩进两个空格

## 5. 代码格式规范

### 5.1 行内代码
使用反引号包裹实现行内代码：`function example()`

### 5.2 代码块
使用三个反引号包裹实现代码块，并指定语言类型：
```javascript
function example() {
  console.log('Hello world');
}
```

### 5.3 代码格式要求
- 代码块必须包含语言类型标识
- 代码应保持正确的缩进和格式
- 避免在代码块内使用不相关的注释

## 6. 表格格式规范

```markdown
| 表头1 | 表头2 | 表头3 |
|-------|-------|-------|
| 单元格1 | 单元格2 | 单元格3 |
| 单元格1 | 单元格2 | 单元格3 |
```

### 6.1 表格格式要求
- 表格内容必须居中或对齐
- 表头与内容分隔线必须使用`|-----|`格式
- 表格列宽可以根据内容调整
  - 左对齐：`|:----|`
  - 居中对齐：`:----:`
  - 右对齐：`|----:`

## 7. 链接与引用格式

### 7.1 内部链接
```markdown
[链接文本](链接地址)
```

**示例：**
```markdown
[开发指南](../06-开发指南/开发指南.md)
```

### 7.2 外部链接
```markdown
[链接文本](URL)
```

### 7.3 引用块
```markdown
> 引用内容1
> 引用内容2
```

## 8. 图片格式规范

```markdown
![图片说明](图片路径)
```

**图片格式要求：**
- 图片必须包含说明文本
- 图片路径应使用相对路径或可访问的URL
- 图片大小应适当，避免过大
- 优先使用SVG格式的图片

## 9. 文档结构规范

### 9.1 文档基本结构

1. YAML front matter 元数据
2. 一级标题（文档标题）
3. 简短的文档简介
4. 目录（可选，对于长文档）
5. 主要内容（使用H2-H4层级标题组织）
6. 结论或总结部分
7. 参考资料（如有）

### 9.2 文档内容格式要求

- 避免段落分隔规范（空行分隔）
- 文档内容应层次分明，逻辑清晰
- 文档内容应使用简单明了的语言
- 避免在同一行内混合不同的格式元素

## 10. 特殊内容格式规范

### 10.1 警告提示

```markdown
> **警告**：此处为重要的警告信息，需要用户特别注意。
```

### 10.2 注意事项

```markdown
> **注意**：此处为需要用户特别关注的信息。
```

### 10.3 提示信息
```markdown
> **提示**：此处为对用户的建议或辅助说明。
```

## 11. 版本控制与更新规范
- 文档每次更新后必须修改`last_updated`字段
- 文档内容发生变化时必须更新版本号
- 重要修改必须记录在`changelog`字段中
- 文档版本号应与项目版本号保持同步

## 12. 文档审核流程

1. 文档编写完成后进行自检
2. 检查所有链接是否有效
3. 检查格式是否符合规范
4. 提交审核并根据反馈修改

## 13. 常见错误
- 标题层级混乱，随意使用不同级别的标题
- 代码块缺少语言标识
- 表格格式不正确，列不对齐
- 图片缺少说明文本
