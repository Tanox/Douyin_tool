## 开发环境设置

在开始开发前，请按照以下步骤设置开发环境：

### 系统要求

- Node.js 14.x 或更高版本
- npm 6.x 或更高版本
- Git

### 安装步骤

1. 克隆仓库

   ```bash
   git clone https://github.com/SutChan/douyin_tool.git
   cd douyin_tool
   ```

2. 安装依赖

   ```bash
   npm install
   ```

3. 验证安装

   ```bash
   npm run build
   ```

   成功构建后，应该会在`dist`目录生成`douyin_ui_customizer.user.js`文件。

### 开发工作流

1. 修改代码
2. 本地测试
3. 提交代码
4. 创建Pull Request

## 代码风格规范

为了保持代码的一致性和可读性，请遵循以下代码风格规范：

### JavaScript 风格

- 使用ES6+语法
- 变量和函数使用驼峰命名法（camelCase）
- 常量使用全大写加下划线（UPPER_SNAKE_CASE）
- 缩进使用2个空格
- 字符串使用单引号
- 花括号与代码在同一行
- 运算符两侧添加空格
- 每行不超过100个字符

### CSS 风格

- 使用BEM（Block Element Modifier）命名约定
- 属性按字母顺序排序
- 每个选择器使用单独的行
- 属性值前添加空格

### 注释规范

- 为复杂函数和算法添加详细注释
- 使用JSDoc格式为API函数添加文档
- 注释应说明代码的目的和实现思路，而非简单描述代码行为

## 提交指南

### 提交消息格式

提交消息应遵循以下格式：

```
类型(可选作用域): 描述

可选的详细说明

可选的引用Issue
```

### 提交类型

- **feat**: 添加新功能
- **fix**: 修复bug
- **docs**: 文档更改
- **style**: 代码风格更改（不影响功能）
- **refactor**: 代码重构（不添加新功能，不修复bug）
- **perf**: 性能优化
- **test**: 添加或修改测试
- **build**: 构建系统或外部依赖更改
- **ci**: CI配置文件和脚本更改

### 提交消息示例

```
feat: 添加暗黑模式支持

- 实现设置面板的暗黑模式
- 添加暗黑模式切换开关
- 更新默认配置以支持主题设置

Closes #123
```

## Pull Request 流程

1. **创建Pull Request**
   - 填写清晰的标题和描述
   - 引用相关Issue
   - 说明更改的目的和实现方式

2. **代码审查**
   - 项目维护者会审查您的代码
   - 根据反馈进行必要的修改

3. **测试**
   - 确保您的更改通过基本测试
   - 验证不会引入新的问题

4. **合并**
   - 审查通过后，项目维护者会合并您的Pull Request
