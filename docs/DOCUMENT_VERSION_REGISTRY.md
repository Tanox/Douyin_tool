## 文档更新流程

1. 从主分支创建文档更新分支
2. 进行文档修改，更新YAML front matter中的版本号和更新日期
3. 同时更新此版本登记表
4. 提交Pull Request，包含文档变更说明
5. 通过审核后合并到主分支
6. 打标签标记文档版本（如文档v1.1）

## 版本升级规则

### 主要版本升级 (MAJOR)

- 文档结构发生重大变化
- 核心概念或术语被重新定义
- 大量内容被添加或重写
- 文档格式或标准发生变化

### 次要版本升级 (MINOR)

- 修正文档中的错误
- 添加少量新内容或示例
- 更新过时的信息
- 改进文档的清晰度或组织结构

## 文档版本兼容性说明

本文档库与代码库版本的兼容性：

- 文档版本 1.0 兼容代码版本 1.0.125 及以上
