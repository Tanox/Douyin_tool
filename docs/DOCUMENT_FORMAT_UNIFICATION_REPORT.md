## 文档格式规范

统一后的文档格式规范如下：

1. **YAML Front Matter**：所有文档必须包含标准的YAML front matter，包含版本、更新日期、作者和变更说明
2. **标题层级**：使用标准的Markdown标题层级（#、##、###等）
3. **内容结构**：
   - 主标题（#）
   - 目录或介绍（可选）
   - 内容章节（##）
   - 子章节（###）
   - 代码示例使用```代码块标记
   - 列表使用-或1. 标记

## 文档版本管理

文档版本管理遵循以下规范：

- 版本号格式：`MAJOR.MINOR`
- 主要变更（如结构调整、大幅内容修改）增加MAJOR版本号
- 次要变更（如修正错误、小幅内容更新、格式统一）增加MINOR版本号
- 所有文档更新必须同步更新`DOCUMENT_VERSION_REGISTRY.md`

## 后续维护建议

1. **保持格式一致性**：新增文档必须遵循统一的格式规范，包含YAML front matter
2. **定期更新版本信息**：文档内容发生变化时，及时更新版本号和更新日期
3. **维护索引文件**：确保`INDEX.md`中的链接和版本信息始终保持最新
4. **定期审核**：定期检查文档内容的准确性和时效性，更新过时信息
