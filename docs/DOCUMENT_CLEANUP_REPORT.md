## 版本管理机制

- **版本号格式**：`MAJOR.MINOR`
- **主要版本升级(MAJOR)**：文档结构重大变化、核心概念重定义、大量内容添加或重写
- **次要版本升级(MINOR)**：修正错误、添加少量新内容、更新过时信息
- **文档更新流程**：创建分支→修改文档→更新版本登记表→提交PR→合并→打标签
